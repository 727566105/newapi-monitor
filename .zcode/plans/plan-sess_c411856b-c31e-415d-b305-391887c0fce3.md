
# NewAPI Monitor macOS 客户端 — 实施计划

## 概述
根据已批准的设计文档，从零开始创建 macOS SwiftUI 原生客户端项目。项目目标：macOS 14+，使用 `@Observable`、Swift Charts、NSStatusItem + NSPopover。

## 实施步骤

### Step 1: 创建 Xcode 项目结构
创建完整的目录结构和项目配置文件：
```
NewAPIMonitor/
├── NewAPIMonitor.xcodeproj/        # Xcode 项目配置
├── NewAPIMonitor/
│   ├── App/
│   │   ├── NewAPIMonitorApp.swift   # App 入口
│   │   └── AppDelegate.swift        # NSStatusItem + NSPopover
│   ├── Models/
│   │   ├── UserInfo.swift
│   │   ├── APIModel.swift
│   │   ├── LogEntry.swift
│   │   ├── QuotaData.swift
│   │   ├── TokenUsage.swift
│   │   ├── Stat.swift
│   │   ├── APIResponse.swift
│   │   └── AppConfiguration.swift
│   ├── Services/
│   │   └── NewAPIClient.swift       # API 客户端 @Observable 单例
│   ├── Views/
│   │   ├── ContentView.swift        # NavigationSplitView 主框架
│   │   ├── SettingsView.swift
│   │   ├── OverviewView.swift
│   │   ├── ModelsView.swift         # Admin only
│   │   ├── StatisticsView.swift
│   │   └── MenuBarPopoverView.swift
│   ├── Helpers/
│   │   ├── DateHelper.swift         # 时间戳转换、聚合
│   │   └── QuotaFormatter.swift     # 配额格式化
│   └── Assets.xcassets/
│       ├── AccentColor.colorset/
│       └── AppIcon.appiconset/
```

### Step 2: 数据模型层 (Models/)
实现所有 Codable 数据结构：
- `UserInfo` — 用户信息（含 role 字段，用于自动检测角色）
- `APIModel` — 模型（status 为 var，支持乐观更新）
- `LogEntry` — 日志条目
- `QuotaData` — 小时粒度用量数据
- `TokenUsage` — Token 配额信息
- `Stat` — 聚合统计
- `APIResponse<T>` / `PaginatedResponse<T>` — 通用响应包装
- `AppConfiguration` — 配置持久化（UserDefaults）

### Step 3: API 客户端层 (Services/)
实现 `NewAPIClient`（@Observable 单例）：
- 配置管理（baseURL、token、role、refreshInterval）
- URLSession + async/await 网络请求
- 所有 API 端点封装：
  - `fetchUserInfo()` — 角色自动检测
  - `fetchTokenUsage()` — 配额查询
  - `fetchStat()` — 聚合统计
  - `fetchLogs()` — 日志分页查询
  - `fetchModels()` / `searchModels()` — 模型列表
  - `updateModelStatus()` — 模型状态切换
  - `fetchQuotaData()` — 小时粒度统计数据
- 错误处理（网络/认证/服务器错误）
- 后台定时刷新 Timer

### Step 4: 辅助工具 (Helpers/)
- `DateHelper` — Unix 时间戳 ↔ Date 转换、按日/周聚合 QuotaData
- `QuotaFormatter` — 配额数值格式化（52K、1.2M）

### Step 5: App 入口 + AppDelegate
- `NewAPIMonitorApp.swift` — SwiftUI App 协议，`@NSApplicationDelegateAdaptor`
- `AppDelegate.swift` — NSStatusItem 创建、NSPopover 管理、点击切换逻辑
- 首次启动检测 → 自动弹出设置页

### Step 6: 主框架 + 设置页
- `ContentView.swift` — NavigationSplitView 侧边栏（根据角色动态显示）
- `SettingsView.swift` — 服务器地址、Token、角色、刷新间隔、连接测试

### Step 7: 概览页
- `OverviewView.swift` — 配额卡片（已用/剩余/RPM）、进度条（含无限额处理）、最近调用表格
- 离线/空数据状态设计

### Step 8: 模型管理页 (Admin Only)
- `ModelsView.swift` — 模型列表 + Toggle 开关 + 搜索（300ms 防抖）
- 乐观更新 + 失败回滚 + Toast 提示

### Step 9: 统计页
- `StatisticsView.swift` — 时间范围选择（1天/7天/30天）+ 模型筛选
- Swift Charts 折线图/柱状图
- 聚合策略：1天=小时粒度，7/30天=按日聚合
- 用量明细表格

### Step 10: 菜单栏下拉视图
- `MenuBarPopoverView.swift` — 配额百分比/进度条、已用/总额/RPM、打开主窗口/设置按钮
- 无限额处理（∞ 无限额度）

### Step 11: 编译验证
- 使用 `xcodebuild` 编译项目
- 修复编译错误
- 确保所有文件正确关联

## 技术要点
- **macOS 14+** 最低版本，使用 `@Observable` 宏
- **无第三方依赖**，全部使用系统框架
- **Swift Charts** 用于统计图表
- **NSStatusItem + NSPopover** 用于菜单栏
- **UserDefaults** 用于配置持久化
- **async/await** 用于网络请求
- **角色自动检测**：启动时调用 `/api/user/self`，403 时降级为 User 角色
